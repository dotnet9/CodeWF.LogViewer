# CodeWF.Log 核心设计

## 目标

CodeWF 使用一个完整、不可变的 `CodeWFLogEvent` 驱动所有输出：

- File 使用独立 `OutputTemplate`，适合保留诊断消息、结构化属性、Scope、Activity 和异常。
- Console、Avalonia `LogView` 和通知严格共享同一个 `LineTemplate`。
- `Message` 是诊断消息；`UserMessage` 是 CodeWF 特有的可选用户友好消息。模板 `{UserMessage}` 为空白时回退到 `{Message}`。
- 组件不做隐式“安全字段”裁剪；调用方把详细字段加入 `LineTemplate` 即表示允许其出现在界面和通知中。

## 核心模型

| 类型 | 职责 |
| --- | --- |
| `CodeWFLogEvent` | 完整日志事件，包含 Message、UserMessage、EventId、Properties、Scopes、Activity 和异常快照 |
| `LogExceptionInfo` | 有界、不可变的异常快照，不长期持有原始 Exception 对象图 |
| `LoggerHost` | 有界队列单消费者，分配 Sequence 并按顺序协调各输出 |
| `LogEventFeed` | 完整事件的有限最近记录、原子重放和非阻塞多订阅广播 |
| `ILineTemplateController` | Console、LogView、通知共享模板的运行时原子更新入口 |
| `IFileOutputTemplateController` | File 独立模板的运行时原子更新入口 |
| `CodeWFLogHealth` | 队列丢弃总数及各日志级别计数 |
| `FileLogSink` | 持久文件流、Flush、按日期/大小滚动和保留清理 |
| `ConsoleLogSink` | 使用共享 `LineTemplate` 输出所有达到阈值的事件 |

## 数据流

```text
ILogger<T> / Logger.*
        │
        ├─ 调用线程：格式化 Message，快照 State/Scope/Activity/Exception
        ▼
有界 Channel<LogCommand>
        │
        ▼
LoggerHost 单消费者（分配 Sequence）
        ├─ FileLogSink：File.OutputTemplate
        ├─ ConsoleLogSink：共享 LineTemplate
        └─ LogEventFeed：同一完整 CodeWFLogEvent
               ├─ LogView：共享 LineTemplate + 独立级别范围
               └─ LogNotifications：共享 LineTemplate + MinimumLevel
```

`Logger.*ToFile` 使用同一队列但只路由到 File。Sequence 对应单消费者实际处理顺序。慢 Feed 订阅者在独立调度中串行接收，不阻塞日志 Pipeline。

## 生命周期与多 Host

- 静态 API 通过 `Logger.Initialize` 创建进程级 Pipeline，`FlushAsync` 是队列屏障，`ShutdownAsync` 排空并释放资源后允许重新初始化。
- MEL Provider 为每个 DI 容器创建独立 `LoggerHost`、`LogEventFeed` 和两个模板 Controller，多个 Host 配置和生命周期互不影响。
- `BridgeStaticLogger=true` 时，第一个可用 Provider 可让 legacy `Logger.*` 路由到该实例；Provider 仍拥有 Pipeline 生命周期，释放时自动解除桥接。
- MEL 全局和 Category 过滤由 `Logging:LogLevel` / `Logging:CodeWF:LogLevel` 负责；File、Console 只保留各自 Sink 阈值。

## 文件与队列策略

- 文件名为 `Log_yyyy_MM_dd.log`，同日按大小追加 `_1`、`_2`。
- 单文件默认上限 1000 MB，默认严格保留最近 30 天；默认不限制目录总容量和文件数量。
- 只有显式配置 `RetainedFileCountLimit` 或 `MaxDirectorySizeBytes` 时，才允许提前删除 30 天内最旧文件。
- 队列支持 `Wait`、`DropNewest`、`DropTraceAndDebug`；非 Wait 模式受 `EnqueueTimeout` 限制。
- 丢弃不会递归写回日志队列，通过 `CodeWFLogHealth` 独立观测。

## Avalonia

- Application 通过 `LogContext.Source` 设置全局 `LogEventFeed`；`LogView.Source` 可局部覆盖。
- Application 通过 `LogContext.LogDirectory` 设置右键打开的全局日志目录；`LogView.LogDirectory` 可局部覆盖，目录解析不依赖静态 Logger 是否初始化。
- LogView 默认 `MinimumLevel=Information`、`MaximumLevel=Critical`，支持有限显示、批量 UI 刷新、独立清空边界和模板更新后重绘。
- Notification 默认关闭；启用后默认 `MinimumLevel=Error`，只订阅新事件。
- InApp 默认最多同时显示 3 条；DesktopWindow 复用一个桌面右下角窗口；展示队列默认容量 100，溢出数量合并提示。

完整 MEL 适配、配置、Demo 和验收标准见 [Microsoft.Extensions.Logging 适配设计](doc/Microsoft.Extensions.Logging适配设计.md)。
